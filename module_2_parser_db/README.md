# Модуль 2: Парсер и Векторная БД

Модуль отвечает за извлечение данных из сырых PDF-файлов, их семантическую обработку и предоставление REST API для мультиагентной системы.

Используем **MinerU** для парсинга.

## Стек технологий
* **Пакетный менеджер:** `uv`.
* **ML / Парсинг:** MinerU.
* **Векторная БД:** ChromaDB.
* **Бэкенд:** FastAPI, Pydantic.
* **Окружение:** Docker (изолированный контейнер с доступом к общему Volume `/data/`).

## Зоны ответственности внутри модуля
* **Арсений (ML & Парсинг):** Скрипты `extractor.py`, `equations.py`. Настройка MinerU, извлечение двухколоночного текста, таблиц и формул. Физическое сохранение кропов изображений в `/data/images/`. Валидация математики.
* **Рома (БД & API):** Скрипты `chunker.py`, `chroma_store.py`, `api.py`. Семантический чанкинг по границам логических абзацев, привязка сохраненных картинок к метаданным текстовых чанков, логика работы ChromaDB и написание эндпоинтов.

## Архитектура и Контракт данных
Точка соприкосновения внутри модуля - `schemas.py`. 
Парсер собирает извлеченные данные в строгий Pydantic-объект `ParsedDocument` (иерархия заголовков, массивы абзацев, пути к графике). 
Скрипты БД принимают `ParsedDocument`, нарезают его на объекты `DBChunk` и сохраняют в ChromaDB. Метаданные чанков хранятся исключительно в виде плоских строк.

## Взаимодействие со смежными модулями

### Для Модуля 1
* Скачанные PDF-файлы должны складываться в расшаренную папку `/data/pdfs/`.
* Обязательно формирование JSON-файла с маппингом и полными метаданными (DOI, название статьи, авторы, журнал, путь к скачанному файлу). Без метаданных парсер не начнет обработку.

### Для Модуля 3
Вам не нужно взаимодействовать с PDF или сырыми JSON-файлами напрямую.
Вам доступен REST API (FastAPI):
* **GET `/api/search?query=...`** — принимает текстовый запрос от вашего агента, осуществляет семантический поиск по векторной базе и возвращает релевантный контекст.
* В ответе API вместе с текстом всегда будут передаваться метаданные `linked_images` — пути к вырезанным картинкам и таблицам для последующей вставки в итоговый LaTeX-шаблон.
* **Важно:** на вашей стороне обязательно должен быть реализован агент-фактчекер. При парсинге могут теряться разделители (например, 110.0 превращается в 110000). Валидатор должен сверять цифры и концентрации из вашей генерации с сырыми данными из нашего API. (может получиться исправить нам)
